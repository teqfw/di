# Code Components (`./ctx/docs/code/components/`)

Path: `./ctx/docs/code/components/AGENTS.md`

## Purpose

The `ctx/docs/code/components/` directory defines implementation-level contracts of core components. Documents at this level specify component interfaces, behavior, responsibility boundaries, and conformance limits.

## Level Boundaries

This level contains component contracts only. It does not define generic coding conventions, source-tree layout rules, testing structure policy, architectural semantics, product meaning, or agent operating procedures.

## Level Map

- `AGENTS.md` — this document, defining the purpose, boundaries, and navigational map of the `components/` level.
- `container.md` — container implementation contract.
- `depid.md` — canonical dependency identity (`DepId`) implementation-level invariants and rules.
- `parser.md` — parser implementation contract.
- `resolver.md` — resolver implementation contract.

## Summary

`ctx/docs/code/components/` provides implementation contracts for the core components and fixes their normative behavioral and interface boundaries.
