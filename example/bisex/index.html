BiSex Sample for JS Code

sources

Overview

This is a demo of the JS code (not TS!) that runs both in a browser and as a Node.js application without transpilation. Therefore, I called this sample as 'BiSex'.

There is one only static import in the code - for DI container in the `main.mjs` script. All other sources are injected in the runtime:

export default class Sample_App {
    /**
     * @param {Sample_App_Config} config
     * @param {Sample_App_Service} service
     */
    constructor(
        {
            Sample_App_Config: config, // default export is an object (singleton)
            Sample_App_Service$: service, // construct a singleton from default export
        }
    ) {...}
}

Running

Open browser console to see console log. Expected output:

Sample App: Creating logger for 'Sample App'.
Sample App: Service 'Sample_App_Service' is created for app 'Sample App'.
The BiSex App: Service running with 'Running...' param.
